# Правила написания контента для гайдов KITON

Использовать единые правила при всех правках и новых разделах базы знаний.

**Источник контента:** все правки и новые файлы базы знаний — только в папке **docs1** (корень). Папку docs1/docs не редактировать; при необходимости синхронизацию выполняете отдельно.

## Понятность

- Чёткие формулировки, по 2-4 предложения на абзац. Одна мысль — один абзац.
- Избегать длинных перечислений без подзаголовков. Разбивать списки на логические блоки с заголовками.

## Без воды

- Не дублировать общие фразы («KITON — это современный сервис…») в каждом разделе.
- Вводное описание роли и возможностей — один раз в начале гайда для этой роли.

## Пояснения там, где нужно

- Для неочевидного (персональный код, связь ученик–родитель, «Газики», исключения из расписания) добавлять 1–2 предложения «зачем это» перед инструкцией.
- Не перегружать очевидные шаги лишними объяснениями.

## Соответствие продукту

- Описывать только то, что есть в KITON.
- Если у других сервисов есть «кошельки» или «абонементы», а в KITON — другая модель (оплаты по ученику, тарифы подписки), формулировать под KITON.
- Не обещать функционал, которого нет (смена пароля, мультивалюта, отдельные кошельки и т.п.). Если функционала вообще нет, то не нужно об этом говорить. 

## Один тон

- Обращение «вы» / «ваш», нейтрально-дружелюбно.
- Без сленга и лишних восклицаний.

## Объём

- На каждый вопрос формата FAQ — один короткий ответ (2–4 предложения), при необходимости 1–2 предложения пояснения.
- Новые блоки — по 3–7 абзацев, без дублирования общих вступлений.
